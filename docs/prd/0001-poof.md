# PRD 0001 - Poof v1 (local MVP)

Zero-knowledge, ephemeral clipboard sharing on Cloudflare Workers + Durable Objects. Paste text, code, an image, or a file; get a short-lived link; the server only ever holds ciphertext.

This PRD synthesizes the grilled design. It does not restate the ADRs; it references them. Read `CONTEXT.md` for vocabulary and `docs/adr/0001`..`0008` for the decisions and their rationale.

## Why

The open-source pastebin/secret-sharing space is crowded (PrivateBin, 0bin) but nothing delivers that experience natively on Workers + Durable Objects, and Firefox Send (the closest "share a thing, link expires" tool) is dead. The gap: a CF-native, opinionated, extremely simple, honest-about-risk tool that deploys in one step and that anyone can self-host on a free Cloudflare account.

## Principles

- **Zero-knowledge** (ADR-0001): the server only ever holds ciphertext. The Link (with its Fragment Key) is the secret.
- **Anonymous**: no accounts, no PII anywhere (ADR-0007 keeps it that way).
- **Ephemeral by default**: 5-minute TTL, burn after one Reveal (ADR-0002).
- **Honest about risk**: the UI states plainly what is and is not protected, including "lose the link and it is gone forever."
- **Self-hostable on free**: only free-plan primitives (SQLite-backed DO, R2 free tier, free Turnstile). No Enterprise features (ADR-0005).

## In scope (v1)

- Create a Clip of Kind text, code, image, or file via one unified paste/drop input (ADR-0003).
- Client-side AES-GCM encryption; two-blob payload (encrypted metadata + encrypted content) (ADR-0001, ADR-0003).
- Link with Fragment Key in the URL fragment; never sent to the server.
- Reveal-gate landing page: decrypt metadata client-side for a pre-reveal card; explicit Reveal releases content (ADR-0002, ADR-0003).
- Lifecycle: TTL `{5m,1h,1d,7d}` default 5m; reveal budget `{1,3,10,unlimited}` default 1; Burn on whichever hits first (ADR-0002).
- Optional 4-digit PIN: server-side release gate + folded into content key + lockout on wrong attempts (ADR-0004).
- Mandatory free Turnstile on create; Turnstile before each PIN attempt (ADR-0005).
- Storage: one DO per Clip; content inline if ≤ ~1MB else R2; 10MB cap (ADR-0006).
- Owner token: hidden "delete now" affordance (ADR-0008).
- Render on reveal: code highlighting + copy, image preview + copy + download, file download, text + copy (ADR-0003).
- UI: Workers Design System structure + Raycast palette (dark-first, coral accent).

## Out of scope (v1)

- Approval-to-open flow (ADR-0007, rejected).
- Accounts, history, dashboards (ADR-0002, ADR-0007).
- Turnstile ephemeral IDs (ADR-0005, Enterprise-gated).
- Streaming/multipart upload for files > ~25MB (ADR-0006).

## Deep modules (and test coverage)

- **`crypto`** (client + shared): key generation, HKDF hierarchy (metadata key = fragment; content key = fragment + PIN), AES-GCM encrypt/decrypt, encode/decode of the Fragment Key. Deep module, heavily unit-tested.
- **`ClipDO`** (Durable Object): the lifecycle state machine. Two-blob storage, atomic reveal-budget decrement, TTL alarm burn, PIN gate + lockout, owner-token burn, R2 orchestration. Deep module, heavily tested under the Workers pool.
- **Worker API**: thin router over `ClipDO`. Create / metadata / reveal / delete endpoints + Turnstile verification. Integration-tested.
- **Client UI**: create + reveal screens. Validated with Playwright (visual + flow), light unit tests on pure helpers.

## Definition of done (local MVP)

A developer can run `wrangler dev` locally and, end to end:

1. Create a text/code/image/file Clip; receive a Link.
2. Open the Link in a fresh context; see the honest pre-reveal card (Kind, size, reveals remaining) without spending a reveal.
3. Reveal; content decrypts client-side and renders by Kind with a working copy/download.
4. Burn works: a Clip with budget 1 is gone after one Reveal; a Clip past its TTL is gone.
5. PIN-protected Clip: correct PIN reveals; wrong PIN is rejected and locks out after the threshold.
6. Owner "delete now" burns the Clip immediately.
7. Turnstile is enforced on create (local test keys).
8. `vitest run` passes (crypto + ClipDO + API suites).
